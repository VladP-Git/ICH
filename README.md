# ICH



\# Фильмотека Sakila — Поисковая Система



Учебный проект по разработке приложения для поиска информации о фильмах с использованием обезличенных данных. Проект объединяет фронтенд-технологии и гибридную бэкенд-архитектуру на базе реляционной (SQL) и документоориентированной (NoSQL) баз данных.



\## Цель проекта

Создание локального (возможно с последующим переносом на VPS) веб-интерфейса, заменяющего стандартный консольный ввод, для удобного поиска кинокартин, фильтрации по жанрам/годам и логирования пользовательской активности.

##  Стек технологий
*   **Frontend:** HTML5, CSS3, Bootstrap 5 (компоненты карточек, адаптивная сетка, кастомные visually-hidden лейблы для Accessibility).
*   **Шаблонизатор:** Jinja2 (динамический рендеринг, интеллектуальный скользящий диапазон пагинации `< 1 ... 50 ... 101 >`, динамические списки).
*   **Backend:** Python 3.14 (Flask в режиме отладки `debug=True`), профилирование времени выполнения запросов СУБД с помощью модуля `time`.
*   **Реляционная БД:** MySQL (удаленная база данных Sakila), использование параметризованных запросов (`%s`) для защиты от SQL-инъекций, оптимизация дубликатов через `DISTINCT`.
*   **NoSQL БД (Аналитика):** MongoDB (локальный сервер MongoDB Community Server), администрирование через MongoDB Compass. Использование мощного конвейера агрегации (`Aggregation Pipeline`) для фильтрации, группировки и сортировки Топ-5 запросов.
*   **Управление версиями:** Git / GitHub (с разделением на публичный репозиторий кода и приватный для учебных материалов).

##  Реализованные архитектурные фичи (MVP+)
1.  **Автономность фронтенда:** Все файлы Bootstrap и Bootstrap-Icons переведены на локальное хранение в папку `static/` для обхода тайм-аутов внешних CDN-сервисов (jsdelivr/cloudflare).
2.  **Умная пагинация:** Реализован постраничный вывод фильмов (по 10 штук на страницу) с автоматическим сохранением GET-параметров поиска в URL. Ряд страниц автоматически сжимается с использованием многоточий `...` при больших объемах данных.
3.  **Защита аналитики:** Логирование в MongoDB срабатывает строго при инициализации нового поиска (`page == 1`), исключая накрутку счетчиков при простом перелистывании страниц пользователем.
4.  **Безопасность соединений:** Коннекты к СУБД обернуты в безопасные конструкции `try/finally` с предохранительной проверкой инициализации переменных (`if client is not None:`).



Запуск и отладка сайа выполняется при помощи бэкенд-сервера на Python — файл backend/app.py. Сервер компилирует страницы и отдаёт их браузеру.


# Алгоритм запуска:
1. Запуск сервера в PyCharm:
    В терминале PyCharm (должен присутствовать префикс (.venv)) нужно ввести команду запуска главного файла:
    python backend/app.py

2. Открытие ссылки:
    В консоли появится лог запуска Flask, где будет написан постоянный адрес. Достаточно удерживая клавишу Ctrl кликнуть мышкой по этой ссылке в терминале:
    * Running on http://127.0.0.1:5000
    * Либо ввести в адресной строке браузера Хром вручную: http://localhost:5000

Отладка «на лету» (Live Reload):
В файле app.py включен режим отладки (debug=True), как только будут внесены изменения в код Python или в HTML-шаблон и нажмете Ctrl + S, сервер Flask в терминале сам мгновенно перезагрузится. Достаточно будет просто обновить страницу в браузере (F5), чтобы увидеть изменения.
